## Getting Started

### First Launch

When you launch the application for the first time:

1. The application creates default directories:
   - `~/YOLOv8_Projects/` - Your projects folder
   - `~/YOLOv8_Models/` - Pre-trained models
   - `~/.yolo_training_tool/logs/` - Application logs
   - `~/.yolo_training_tool/crashes/` - Crash reports
   - `~/.yolo_training_tool/autosave/` - Auto-save state

2. The main window opens with empty workspace
3. Select **File > New Project** to begin

### System Requirements

- **Python**: 3.8 or higher
- **RAM**: 8GB minimum, 16GB+ recommended
- **GPU**: NVIDIA GPU with 6GB+ VRAM (optional but recommended)
- **Disk Space**: 10GB minimum for models and datasets

## Creating a Project

### Step 1: New Project Dialog

Click **File > New Project** (Ctrl+N)

Fill in:
- **Project Name**: e.g., "Defect_Detection"
- **Location**: Where to save (default: ~/YOLOv8_Projects)
- **Description**: (Optional) Brief description

The tool creates this structure:
```
Defect_Detection/
├── images/        # Original images
├── labels/        # Annotation files
├── train/         # Training split
├── val/           # Validation split
├── test/          # Test split
├── models/        # Trained models
├── runs/          # Training runs
└── config.yaml    # Project configuration
```

## Adding Images

### Method 1: Drag & Drop

Simply drag image files or folders into the application window.

### Method 2: File Menu

**File > Import Images**

You can:
- Select individual files
- Select an entire folder (recursively imports all images)

### Supported Formats

- `.jpg`, `.jpeg`
- `.png`
- `.bmp`
- `.webp`
- `.tiff`

### Tips

- Use consistent image quality (similar resolution and lighting)
- Aim for 100+ images per class for good results
- Higher quantity often beats higher quality

## Defining Classes

### Add Classes Dialog

**Edit > Manage Classes**

For each defect/object type to detect:

1. Click **Add Class**
2. Enter class name (e.g., "Crack", "Scratch", "Good")
3. Choose a color (auto-generated by default)
4. Click **OK**

### Best Practices

| Recommendation | Reason |
|----------------|--------|
| Use clear, distinct names | Avoid confusion during annotation |
| Limit to 5-10 classes initially | Easier to debug |
| Start with binary (Good/Defect) | Build complexity gradually |
| Use English class names | Avoids encoding issues |

## Annotating Images

### Bounding Box Mode

1. Select an image from the panel
2. Choose **Box Tool** (B)
3. Click and drag to draw a box around the object
4. Select the class from the dropdown
5. Move to next image (D) or previous (A)

### Polygon Mode

For irregular shapes (segmentation):

1. Select **Polygon Tool** (P)
2. Click points around the object
3. Press **Enter** or double-click to close
4. Select class

### Keyboard Shortcuts

| Action | Shortcut |
|--------|----------|
| Save | Ctrl+S |
| Next image | D |
| Previous image | A |
| Delete annotation | Delete |
| Copy annotation | Ctrl+C |
| Paste annotation | Ctrl+V |
| Undo | Ctrl+Z |
| Redo | Ctrl+Y |
| Zoom in | Ctrl++ |
| Zoom out | Ctrl+- |
| Reset zoom | Ctrl+0 |

### Annotation Tips

✅ **DO**:
- Draw boxes tight around objects
- Annotate ALL instances in each image
- Be consistent with class assignments
- Mark partial objects too

❌ **DON'T**:
- Make boxes much larger than the object
- Skip ambiguous cases (decide and be consistent)
- Mix annotation styles between images

## Splitting Dataset

### Dataset > Split Dataset

Recommended ratios:
- **Train**: 70% (learning)
- **Validation**: 20% (tuning)
- **Test**: 10% (final evaluation)

### Random Seed

The seed ensures **reproducibility**:
- Same seed + same data = same split every time
- Default: 42
- Change only if you want a different split

⚠️ **Important**: Once you split, label files are copied to split folders. Don't manually move files between splits!

## Training a Model

### Training Configuration

Open **Training Window** to configure:

#### Basic Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| Model | YOLOv8s | Model size (n=fastest, x=most accurate) |
| Image Size | 640 | Input resolution |
| Epochs | 100 | Number of training passes |
| Batch Size | 16 | Images per training step |

#### Model Selection Guide

| Model | Speed | Accuracy | Use Case |
|-------|-------|----------|----------|
| YOLOv8n | ⚡⚡⚡⚡⚡ | ⭐⭐ | Real-time, mobile |
| YOLOv8s | ⚡⚡⚡⚡ | ⭐⭐⭐ | **Recommended** balance |
| YOLOv8m | ⚡⚡⚡ | ⭐⭐⭐⭐ | Higher accuracy |
| YOLOv8l | ⚡⚡ | ⭐⭐⭐⭐⭐ | Production accuracy |
| YOLOv8x | ⚡ | ⭐⭐⭐⭐⭐+ | Maximum accuracy |

#### Augmentation Presets

| Preset | When to Use |
|--------|-------------|
| **Light** | Stable lighting, large dataset |
| **Medium** | General purpose (default) |
| **Heavy** | Small dataset, varied conditions |
| **Industrial** | Go/NoGo inspection (recommended for QC) |

#### Advanced Settings

- **Patience**: Stop early if no improvement (recommended: 50)
- **Cosine LR**: Smoother learning rate decay
- **AMP**: Mixed precision training (faster on modern GPUs)
- **Workers**: Data loader threads (auto-detected)

### Pre-flight Checks

Before training starts, the tool automatically checks:
- ✓ Disk space available (need 2GB+)
- ✓ GPU available if specified
- ✓ Configuration is valid
- ⚠️ Warns if batch size may exceed GPU memory

### Starting Training

1. Click **Start Training**
2. Confirm the configuration
3. Training begins in background
4. Monitor progress in real-time

## Monitoring Training

### Metrics to Watch

#### Training Loss
- Should **decrease** over time
- If it plateaus or increases → reduce learning rate

#### Validation Loss
- Should also decrease
- If validation > training loss significantly → overfitting

#### mAP50 (Mean Average Precision @ IoU 0.50)
- **0.0 - 0.5**: Poor
- **0.5 - 0.7**: Acceptable
- **0.7 - 0.9**: Good
- **0.9 - 1.0**: Excellent

#### Precision vs Recall
- **High Precision, Low Recall**: Misses some defects (conservative)
- **Low Precision, High Recall**: Many false positives (aggressive)
- **Both High**: Ideal balance

### Pause / Resume

- Click **Pause** to halt training (saves checkpoint)
- Click **Resume** to continue from last epoch

### Stop Training

- **Stop** ends training and saves best weights
- The model from `runs/train/<exp>/weights/best.pt` is the result

## Testing the Model

### Tools > Test Model

#### Test on Images

1. Load a model (best.pt)
2. Select images to test
3. Set confidence threshold (0.25 default)
4. Click **Run Inference**
5. View:
   - **Text Results**: Detection counts per class
   - **Image Results**: Visual overlay with bounding boxes

#### Test on Videos

1. Select a video file
2. Run inference (frame-by-frame)
3. Get per-class detection statistics

#### Batch Testing

Process multiple images at once:
1. Select folder of images
2. Choose batch size
3. View aggregate statistics

## Tips for Best Results

### Data Collection

1. **Quantity matters**: 200+ images per class
2. **Variety helps**: Different angles, lighting, backgrounds
3. **Real-world conditions**: Match production environment
4. **Edge cases**: Include difficult/ambiguous examples

### Annotation Quality

1. **Be precise**: Tight bounding boxes
2. **Be consistent**: Same rules across all images
3. **Review work**: Random check of annotations
4. **Multiple annotators**: Compare for consensus

### Training Strategy

1. **Start small**: Use YOLOv8n to validate setup
2. **Scale up**: Move to YOLOv8s/m for production
3. **Use Industrial preset** for inspection tasks
4. **Monitor metrics**: Stop if overfitting
5. **Use early stopping**: Patience=50 is good default

### Hardware Tips

| GPU VRAM | Recommended Settings |
|----------|---------------------|
| 4 GB | YOLOv8n, batch=8, imgsz=512 |
| 6 GB | YOLOv8s, batch=16, imgsz=640 |
| 8 GB | YOLOv8m, batch=16, imgsz=640 |
| 12 GB+ | YOLOv8l/x, batch=16-32, imgsz=640+ |

### When Things Go Wrong

#### Model not detecting anything
- Lower confidence threshold (try 0.1)
- Check class IDs match training
- Verify model file is not corrupted

#### Poor accuracy
- More training data
- Check annotation quality
- Try larger model (YOLOv8m or l)
- Increase epochs to 200+

#### Out of memory error
- Reduce batch size (try 8 or 4)
- Reduce image size (512 or 416)
- Use smaller model (YOLOv8n)

#### Training is slow
- Enable AMP (mixed precision)
- Use cache='ram' if dataset fits in RAM
- Reduce image size
- Use smaller model
